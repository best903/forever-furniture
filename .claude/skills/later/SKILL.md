# Later — Backlog 기록 스킬

## 핵심 원칙

**plan.md의 Backlog 섹션에 "나중에 할 것"을 기록한다.** 기록만 하고, 구현하지 않는다.

## 워크플로우

### 1. 입력 파싱
- `$ARGUMENTS`에서 **무엇을** 나중에 할 건지 추출
- 카테고리 자동 판별:
  - 데이터/아키텍처 관련 → `데이터 & 아키텍처`
  - 배포/플랫폼 관련 → `플랫폼 & 배포`
  - UI/UX 관련 → `사용자 경험`
  - 돈/결제 관련 → `수익화`
  - 기능 관련 → `기능 확장`
  - 관리/운영 관련 → `프로젝트 관리`
- 판별 불가 시 `AskUserQuestion`으로 확인

### 2. plan.md 읽기
- `Read`로 plan.md 로드
- Backlog 섹션의 해당 카테고리 위치 파악

### 3. 항목 추가
- `Edit`으로 해당 카테고리 하위에 항목 추가카카오 ㅅ
- 형식: `- [ ] **항목명** — 설명. (출처: {맥락} {오늘 날짜})`
- 중복 체크: 이미 비슷한 항목이 있으면 기존 항목 업데이트

### 4. 확인
- 추가된 내용을 Director에게 한 줄로 보고

## 금지 사항
- 코드 수정 금지
- Backlog 외 섹션 수정 금지
- 항목 삭제 금지 (삭제는 Director가 직접)
